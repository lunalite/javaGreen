About

This Energy Debugger is a java application that generates a simple report of finding the energy consumption of java programs. As of now, this application is in its most basic form with only the power measurement of an application obtained via another machine through powertop.

For more information, please visit this link on github.

Running it

Having installed the application and opening it, you will arrive at the GUI interface which is empty. In order to start running and calculating the energy consumption, go to

[File > Caclulate Energy] [Shortcut: alt+1]

The folder directory will be the root directory where your java package lies, while the main java filename is the file where the main code exists i.e. the start of your entire java application.

As an example, say we have this particular java project:

algorithms
---misc
---===NumGenerator.java
---search
---===BinarySearch.java
---sort
---===MergeSort.java
---===QuickSort.java
---===BinarySort.java
---Main.java

You would point the folder location to where this algorithms package is at, say /Users/abc/IdeaProjects/algorithms/src/algorithms2 and the main java file to /Users/abc/IdeaProjects/algorithms/src/algorithms2/algorithms/Main.java

If the package compiles/decompiles successfully, then the decompiled bytecodes should show on the root GUI, else you will have to check if there is an issue with your package that prevents it from compiling.

After compilation and decompilation is done, the application will push the compiled .jar file into gitlab (currently, this gitlab repository could not be changed) with a jenkins server listening for any changes. This allows for the target.jar file to be pulled into the local machine where powertop is configured, before measurement of energy is done and sent back to the local machine via netcat.

Understanding the values

If everything runs successfully, then you should see on the right colmun, a list of energy measurement. This list of energy measurement is obtained from the number of values that consist of 5-seconds interval, and do include periods of time when the jvm initially started and garbage collector working its magic at the end.

The current unit for measurement as seen is milliwatts (mW). To put things into perspective, a table is shown below of the various values that are running on the processor

Power estimation (mW) Description
8150 Network Interface: enp9s0 (r8169)
2640-2790 Display backlight
1070-1170 USB device: 2.4G Keyboard Mouse
207-297 dbs_work_handler
76.5-130 Laptop fan
6.07-18.1 java app sleeping close to 99% of the 5 seconds

There are currently 12 different values shown on the Energy Debugger Application. The cached version shows all values that have been stored, the new version shows the newest values obtained and the overall version shows all values, including the newest and cached version added up together.

If more information is required, you could export the values using

[file > Export as Csv]

Frequently Asked Questions

  1. What if no decompilation bytecode is shown?

    If not decompilation bytecode is shown, check to see that your java application will be able to compile and run.

  2. 2. What if no energy measurement returns?

    3. If no energy measurement returns, check to see that your program runs for at least a 5-10 seconds. A wrapper is needed to run the program in a while loop is needed if your application finishes running too soon. The shortest iteration currently being measured is at least 5 seconds.
